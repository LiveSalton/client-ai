# 60 - 实现前扫描与主审查

## 何时读取

- 进入代码实现前读取。
- 子 agent 完成后、主 agent 汇总前再读一次。

## 必须产出

- 复用点清单
- 最小修改清单
- 风险清单
- 与已确认 ASCII / 置信度的对照结论
- 审查结论
- 未验证项

## 规则

- 先读当前代码，再做 UI -> Code 映射。
- 只允许对已确认 proposal 中的范围做实现。
- 审查必须对照 ASCII 1:1 复刻稿、ASCII 层级线稿、元素表和置信度表。
- 只改当前需求直接相关的文件、资源和文档。
- 能复用就复用，不能为了“更像”而新造重复结构。
- 不做顺手重构，不碰无关模块，不批量格式化无关文件。
- 主 agent 必须严审子 agent 输出，发现问题先回写当前 change 再继续。

## 审查清单

- [ ] 是否只改了必要文件
- [ ] 是否复用了现有结构
- [ ] 是否符合已确认 proposal
- [ ] 是否覆盖 ASCII 线稿中的全部元素 ID
- [ ] 是否处理所有 low confidence 项
- [ ] 是否清理了临时内容
- [ ] 是否把新发现写回 OpenSpec
- [ ] 是否保留未验证项

## 主审输出模板

```text
reuse: <可复用点>
edit: <实际修改>
match: <与 ASCII / 置信度的对照>
risk: <主要风险>
verify: <已验证 / 未验证>
next: <下一步>
```
