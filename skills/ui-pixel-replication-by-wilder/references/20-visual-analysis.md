# 20 - 视觉语义分析

## 何时读取

- 看到截图、参考图、Figma 画面、局部裁切图时读取。
- 需要做像素级判断、元素分割或图标候选提取时读取。

## 必须产出

- 场景理解
- 区域划分
- 元素清单
- 图标候选
- 布局关系
- 像素参数
- 低置信度项
- 可交给 `25-ascii-confidence.md` 的元素 ID 表

## 分析字段

| 字段 | 要求 |
| --- | --- |
| element_id | 稳定、可追踪、不可随意改名 |
| region_id | 归属区域，便于分片 |
| type | 容器、文本、按钮、图标、图片、分割线、遮罩等 |
| bounds | 坐标和尺寸 |
| style | 颜色、圆角、边框、阴影、透明度、字体、间距 |
| confidence | high / medium / low，并说明依据 |

## SOP

1. 先写画布尺寸、输入来源和主任务。
2. 按首屏、导航、主内容、列表、弹层等区域划分 `region_id`。
3. 给每个可见元素分配稳定 `element_id`，不要后续随意改名。
4. 为每个元素记录 bounds、style、source、confidence。
5. 把图标、图片、头像、状态徽标单独列入候选表。
6. 把缺失、遮挡、模糊、冲突的部分列入 low confidence。
7. 交给 `25-ascii-confidence.md` 生成 ASCII 证据和门禁结果。

## 失败判定

- 只写“整体很像”但没有元素表，失败。
- 漏掉可见元素且没有标 `unknown-*`，失败。
- 元素 ID 在不同表格中不一致，失败。
- 把推断值写成事实，失败。
- 未给 low confidence 下一步动作，失败。

## 输出顺序

1. 场景与主任务
2. 区域划分
3. 元素列表
4. 图标候选
5. 布局与层级
6. 像素样式
7. 低置信度和待确认项
8. 交给 ASCII / 置信度门禁的输入摘要
