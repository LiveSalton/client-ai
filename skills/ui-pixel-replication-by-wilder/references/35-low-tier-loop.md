# 35 - 低级模型多轮循环

## 何时读取

- 低级模型输出泛泛而谈、跳过步骤、提前结束或开始猜测时读取。
- ASCII 证据缺失、置信度未达标或元素表不完整时读取。
- 复杂页面进入编码前读取，用于安排至少两轮复查。

## 目标

把一次长任务拆成可验证的短循环，让低级模型每轮只做一件事，并用固定产物判断是否可以进入下一轮。

## 循环阶段

| 轮次 | 输入 | 任务 | 必须输出 | 通过条件 |
| --- | --- | --- | --- | --- |
| R0 Packet | 原图 / Figma / CSS / 用户约束 | 整理输入包 | 输入清单、冲突、待确认项 | 来源和优先级清楚 |
| R1 Detect | R0 + 原图 | 分区和元素识别 | 区域表、元素表、图标候选 | 可见元素无明显遗漏 |
| R2 Draw | R1 | 画 ASCII 证据 | ASCII 1:1、ASCII 层级线稿 | ID 与元素表一致 |
| R3 Score | R1 + R2 | 打置信度 | 置信度表、blocked 项 | 达到门禁或列出修复项 |
| R4 Repair | R3 blocked 项 | 定点修复 | 修改后的表格和线稿 | blocked 项减少或转人工确认 |

## 循环次数

- 简单页面：至少 1 轮完整 `R1 -> R3`。
- 复杂页面、首屏重写、低级模型验证：至少 2 轮完整 `R1 -> R3`。
- 同一 blocked 项连续 2 轮无法提升时，停止猜测，转为用户确认。
- 达到 `R4` 仍未过门禁时，不得进入实现。

## 每轮提示约束

```text
scope: 只处理 <region/page>
input: 使用当前 OpenSpec change 中的事实，不引入新体系
output: 只输出本轮表格 / ASCII / 置信度，不写代码
fail: 缺证据时写 blocked，不要补脑
```

## 主 agent 复核

- 检查是否有空泛描述替代表格。
- 检查 ASCII ID 是否全部出现在元素表。
- 检查 low confidence 是否有原因和下一步。
- 检查是否在用户确认前开始编码。
- 不通过时只回到对应轮次，不重跑全部流程。
