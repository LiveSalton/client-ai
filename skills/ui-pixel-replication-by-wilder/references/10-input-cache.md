# 10 - 输入与缓存

## 何时读取

- 看到 Figma、截图、CSS、混合设计输入时读取。
- 需要建立工作区、缓存原始输入或整理冲突时读取。

## 必须产出

- 输入清单
- 数据源优先级
- 工作区路径
- 原始缓存位置
- 冲突表
- 低置信度项

## 输入优先级

1. Figma 结构化数据
2. CSS / 样式导出
3. 截图 / 图片的 VLM 分析
4. 用户文本

## 规则

- Figma 原始响应必须整块缓存，原始数据不按 chunk 作为持久存储单位。
- 派生摘要、索引、冲突和解析结果要单独记录，不得覆盖原始缓存。
- CSS 用于几何和样式校准，不替代视觉语义分析。
- 输入冲突必须显式记录，不能静默改值。
- 当高优先级输入缺失关键字段时，才允许低优先级输入补齐，并标注为推断值。

## 最小检查清单

- [ ] 是否已经确认当前页面工作区
- [ ] 是否保存了原始输入
- [ ] 是否分清了事实和推断
- [ ] 是否把冲突写回当前 change
