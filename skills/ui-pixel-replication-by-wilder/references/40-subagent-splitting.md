# 40 - 子 agent 拆分

## 何时读取

- 需要开始实现、分配任务或恢复上下文时读取。
- 发现单个模型上下文装不下当前复刻任务时读取。

## 必须产出

- 子 agent 列表
- 每个 agent 的文件 / 模块所有权
- 每个 agent 的输入和输出
- 交接顺序
- 主 agent 审查点
- 编码是否已获用户确认

## 规则

- 子 agent 最多 6 个。
- 每个子 agent 只能修改自己负责范围内的内容。
- 子 agent 之间不相互覆盖、不回写彼此已经完成的内容。
- 主 agent 只负责调度、整合和最终审查，不抢写具体实现。
- 任何新发现都要回到当前 OpenSpec change。
- 用户确认 proposal 前，不得启动编码子 agent。
- 分析 / 复查类子 agent 只能输出表格、ASCII、置信度和问题，不写代码。
- 编码子 agent 只接收已确认 proposal、ASCII 证据、置信度表和自己的文件所有权。

## 推荐分片

| 分片 | 典型职责 |
| --- | --- |
| 1 | 输入缓存与冲突整理 |
| 2 | 视觉语义分析与元素表 |
| 3 | ASCII 证据、置信度评分与低级模型循环 |
| 4 | 规格 / OpenSpec 回写 |
| 5 | 资产、图标、代码实现或 UI 映射 |
| 6 | 主审查与收敛 |

## 交接模板

```text
agent: <name>
owner: <file/module>
stage: <analysis|spec|implementation|review>
input: <需要的上下文>
output: <交付物>
done: <完成条件>
```

## 保护条款

- 上下文不够时先停，不要靠猜。
- 某个分片完成后先落盘，再开下一个分片。
- proposal 未确认时，任何 agent 输出中不得包含代码补丁。
- 主 agent 审查前，不宣布完成。
